# 产品需求文档（PRD）：每日消费记录助手 V1.0.1

## 1. 文档概述

- **产品名称**：每日消费记录助手
- **版本号**：V1.0.1（增量版本）
- **文档状态**：初稿
- **编写日期**：2026/06/11
- **基于版本**：V1.0
- **变更说明**：新增用户注册与登录功能，实现多用户数据隔离
- **目标平台**：Web 网页端（响应式设计，优先适配手机浏览器体验）
- **视觉风格**：粉嫩、可爱、治愈系
- **后续规划**：V2.0 迭代原生 App（iOS/Android），V3.0 覆盖微信小程序，实现多端数据同步。

## 2. 产品定位与背景

### 2.1 产品背景
V1.0 版本上线后，用户反馈希望能够**在不同设备间同步数据**，且希望有个人账户来保护隐私。V1.0.1 在保留本地存储优势的同时，引入简单的用户系统，为后续 V2.0 云端同步奠定基础。

### 2.2 产品定位
打造一款**极简、轻量、无打扰、超可爱**的个人 Web 端记账工具。通过粉嫩的视觉语言和流畅的交互，让记账这件"反人性"的事变得轻松治愈。

### 2.3 目标用户
- **核心用户**：18-30 岁年轻群体，有日常记账习惯或希望培养记账习惯的个人用户。
- **使用场景**：每日消费后 30 秒内快速补记；月末回顾消费构成；多设备间使用同一账户。

### 3.2 详细功能说明

#### 3.2.1 用户模块（新增）

| 功能点 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| **注册页面** | 包含用户名输入框、密码输入框、确认密码输入框。用户名 3-20 位，密码 6-20 位。点击注册后校验格式，校验通过则创建账户并跳转至记账首页。 | P0 |
| **登录页面** | 包含用户名输入框、密码输入框、登录按钮。点击登录后校验账户存在且密码正确，校验通过则进入记账首页。校验失败提示“用户名或密码错误”。 | P0 |
| **会话管理** | 登录成功后，将用户 ID 存入 `localStorage`（键名 `current_user_id`）。每次打开应用时，自动检测是否有已登录用户，若有则直接跳转至记账首页。 | P0 |
| **登出功能** | 在"我的"页面提供"退出登录"按钮，点击后清除 `current_user_id`，跳转至登录页面。 | P0 |
| **数据隔离** | 用户的消费记录、分类数据与用户 ID 绑定。查询时仅返回当前登录用户的记录，确保数据隐私。 | P0 |

**交互说明**：
- 注册/登录页面背景为粉色渐变，配有小动物插画。
- 输入框获得焦点时，粉色边框高亮。
- 按钮点击时缩放动画（`scale(0.96)` → `scale(1)`）。
- 错误提示使用 Toast 形式，3 秒后自动消失。

## 4. 非功能需求

### 4.1 性能需求
- **首屏加载**：在 4G 网络下，首屏加载时间不超过 2 秒。
- **交互响应**：用户点击、滑动操作反馈延迟不超过 100ms。
- **数据容量**：每用户支持 5 年内每日 20 笔记录的存储空间而不出现明显卡顿。

### 4.2 兼容性需求
- **浏览器兼容**：
  - 移动端：iOS Safari (iOS 13+)、Chrome for Android (Android 9+)、微信内置浏览器。
  - PC 端：Chrome、Edge、Safari 最新两个主版本。
- **屏幕适配**：采用响应式布局，核心适配 375px - 430px 宽度的手机屏幕，同时兼容平板和 PC 端浏览（居中卡片式布局，限制最大宽度 600px）。

### 4.3 安全与隐私
- **密码存储**：密码在本地存储前进行 **SHA-256 哈希处理**，不存储明文密码。服务端暂无（V2.0 引入），此处为客户端校验逻辑。
- **会话安全**：`current_user_id` 存储于 `localStorage`，与用户密码分离。
- **数据隔离**：所有数据查询均需带上当前用户 ID，仅返回属于该用户的数据。
- **隐私告知**：在首次引导和"个人中心"中，明确提示用户数据存储位置与安全建议。

### 4.4 可用性需求
- **离线可用**：支持 PWA（Progressive Web App）基础配置，允许用户"添加到主屏幕"，在无网络环境下也能正常记账和查看。
- **多设备提醒**：在"我的"页面提示"当前数据仅保存在本设备，换设备登录需重新记录"。

## 6. 数据模型（前端本地存储）

### 6.1 用户 (User)

```typescript
interface User {
  id: string;          // 唯一标识，UUID
  username: string;    // 用户名，3-20 位，唯一
  passwordHash: string; // SHA-256 哈希后的密码
  createdAt: number;   // 注册时间戳
}
```

### 6.2 消费记录 (Record)

```typescript
interface Record {
  id: string;          // 唯一标识，UUID
  userId: string;     // 关联的用户 ID（数据隔离关键字段）
  amount: number;      // 金额，单位元，保留两位小数
  type: 'expense';    // V1.0 仅支持 expense
  categoryId: string;  // 关联的分类 ID
  remark: string;      // 备注，默认空字符串
  date: string;        // 日期，格式 YYYY-MM-DD
  createdAt: number;   // 创建时间戳
  updatedAt: number;   // 更新时间戳
}
```

### 6.3 消费分类 (Category)

```typescript
interface Category {
  id: string;          // 唯一标识
  userId: string;      // 关联的用户 ID（自定义分类归属）
  name: string;        // 分类名称
  icon: string;        // 图标名称/Emoji
  type: 'expense';     // V1.0 仅支持 expense
  isCustom: boolean;    // 是否为用户自定义
  createdAt: number;   // 创建时间戳
}
```

### 6.4 存储键名规范

| 数据类型 | 键名格式 | 说明 |
| :--- | :--- | :--- |
| 用户列表 | `expense_users_v1` | JSON 数组，存储所有注册用户 |
| 当前登录用户 | `expense_current_user_v1` | 字符串，存储当前登录用户 ID |
| 消费记录 | `expense_records_v1` | JSON 数组，存储当前用户的记录 |
| 消费分类 | `expense_categories_v1` | JSON 数组，存储当前用户的自定义分类 |
| 连续打卡 | `expense_streak_v1` | 数字，当前用户的连续打卡天数 |
| 最后打卡日期 | `expense_last_date_v1` | 字符串，格式 YYYY-MM-DD |

## 8. 技术要求与约束

- **前端框架**：建议采用 React + Vite 或 Vue 3 + Vite 进行开发，配合 TypeScript。
- **UI 组件库**：移动端优先，建议使用 Vant (Vue) 或 Ant Design Mobile (React)。
- **图表库**：统计页甜甜圈图建议使用 ECharts 或 Chart.js。
- **本地存储**：使用 `localStorage` 存储所有数据，通过 `localForage` 或 `Dexie.js` 封装 IndexedDB 支持大数据量。
- **加密**：密码使用 **SHA-256** 哈希处理（通过 Web Crypto API 或第三方库如 `js-sha256`）。
- **构建与部署**：构建产物为纯静态文件，可部署至 GitHub Pages / Vercel / Netlify。

## 10. 风险与应对

| 风险点 | 描述 | 应对措施 |
| :--- | :--- | :--- |
| **密码明文风险** | V1.0.1 密码仅存于本地，若设备被他人访问，密码有被推断风险。 | 1. 使用 SHA-256 哈希存储；2. 在"我的"页面提示用户设置强密码；3. V2.0 尽快上线服务端认证（JWT/Session）。 |
| **多设备数据不同步** | 当前版本数据仅保存在本地，换设备后数据丢失。 | 1. 在"我的"页面明确提示数据不同步的风险；2. 提供数据导出功能供备份；3. V2.0 上线云同步。 |
| **用户名重复注册** | 同名用户无法注册，但前端校验在注册时才知道。 | 1. 注册时实时校验用户名是否已存在；2. 若重复，引导用户尝试类似用户名或直接登录。 |
